# [제품 요구사항 정의서] watchdog-hq: 제품 요구사항 정의서 (PRD)

본 문서는 상용 SaaS 가용성 관제 서비스 `watchdog-hq` 개발의 범위와 비즈니스 성공 지표, 기능별 우선순위를 정의하여 최소 기능 제품(MVP)의 런칭 스코프를 확정하기 위한 제품 요구사항 정의서(Product Requirement Document)입니다.

---

## 1. 제품 목적 및 성공 지표 (Objectives & KPIs)

### 1.1 해결하고자 하는 문제 (Problem Statement)
1. **슬랙을 안 쓰는 소상공인:** 개인 쇼핑몰이나 소상공인은 슬랙, 디스코드 같은 개발용 메신저를 사용하지 않아 이메일 외에는 실시간 장애를 인지할 대안이 없음.
2. **비싼 해외 알림 서비스:** 글로벌 강자(UptimeRobot 등)는 국내로 SMS나 전화를 발송할 때 단가가 매우 비싸고 발송 성공률이 현저히 떨어짐.
3. **불편한 해외 결제:** 달러 결제로 인한 외화 수수료 발생 및 국내 간편결제(네이버페이, 카카오페이 등)의 부재로 구독 이탈율 발생.

### 1.2 비즈니스 성공 지표 (Success Metrics)
* **목표 가입자:** 출시 3개월 이내 유효 가입자 수 1,000명 돌파.
* **유료 전환율:** 무료(Free) 사용자의 유료 플랜(Starter/Pro) 구독 전환율 3% 이상 유지.
* **알림 신뢰성:** 알림 발송 지연 시간 평균 5초 이내, 발송 유실률 0.1% 미만.

## 2. 사용자 스토리 (User Stories)

* **페르소나 A (온라인 셀러):** 
  "나는 쇼핑몰을 운영하는 비개발자 점주다. 사이트가 느려지거나 서버가 닫혔을 때, 내가 항상 들여다보는 **카카오톡으로 즉시 알림**을 받아 마케팅 광고 집행을 즉시 중단하고 대행사에 복구를 지시하고 싶다."
* **페르소나 B (마케터):** 
  "나는 브랜드 랜딩 페이지 광고 캠페인 담당자다. 퇴근 후나 주말에 랜딩 페이지 서버가 트래픽 초과로 죽었을 때, **문자나 슬랙으로 알림**을 받아 내 광고 예산이 무의미하게 허공으로 날아가는 것을 방지하고 싶다."
* **페르소나 C (1인 풀스택 개발자):** 
  "나는 여러 외주 사이트를 관리한다. 내가 만든 대시보드가 타인에게 무단 조작되지 않도록 **관리자 세션을 검증**하고 싶고, 일반 대중에게 주소를 공유할 때는 **상세 URL 정보가 가려진 채 가용 상태만 공개**되도록 하고 싶다."

## 3. 요구사항 정의 및 기능 우선순위 (Prioritization)

성공적인 린(Lean) 런칭을 위해 기능 요구사항을 P0(필수), P1(다음 배포), P2(선택 사항)로 구분합니다.

### 3.1 P0: MVP 필수 기능 (Must-have)
이 기능들이 구현되지 않으면 제품을 시장에 런칭하지 않습니다.

* **[회원 가입/인증]:** 구글 및 깃허브 소셜 로그인 연동 (Next-Auth / JWT 세션 인증).
* **[가용성 모니터링 관리 (Admin)]**:
  * 헬스체크 대상 등록/삭제 기능.
  * 체크 주기(Interval) 및 타임아웃(Timeout) 세부 설정 기능.
* **[보안 및 마스킹]**:
  * 쿼리 파라미터(`/?token=...`)를 활용한 초경량 관리자 세션 쿠키 발급.
  * 비인가 방문자 접근 시 API 등록/삭제 버튼 비활성화 및 URL 상세 마스킹 처리.
* **[기본 알림 연동]**: 이메일 알림 및 슬랙/디스코드 수신용 Incoming Webhook 연동.
* **[로컬 특화 알림]**: 솔라피(Solapi) 또는 뿌리오 API를 통한 카카오 알림톡 및 국내 SMS 연동.
* **[구독 및 결제]**: Stripe 및 토스페이먼츠 연동을 통한 월 정기 구독 결제 모듈 및 요금제 등급별 등록 제한(2개, 10개, 50개) 기능.

### 3.2 P1: 차순위 기능 (Should-have)
MVP 런칭 성공 후 1~2개월 이내에 정기 업데이트로 추가할 스펙입니다.

* **[공개 상태 페이지 (Status Page)]**: 유료 고객용 독립 도메인 매핑 및 공개용 Uptime 상태 보드 스킨 제공.
* **[SSL 인증서 감시]**: 헬스체크 시 SSL 인증서 잔여 만료일을 계산하여 30일/7일 전 이메일/알림톡 발송 기능.
* **[다중 리전 교차 체크]**: 최소 2곳 이상의 분산 노드(서울, 미국 등)에서 교차 체크하여 모두 실패했을 때만 장애 판정을 내리는 False Alert 필터링 기능.

### 3.3 P2: 장기 로드맵 기능 (Nice-to-have)
비즈니스 스케일업 단계에서 도입을 검토할 항목입니다.

* **[전화 알림]**: TTS(Text-to-Speech) API를 연동하여 서버 다운 시 전화를 걸어 경고해주는 음성 안내 기능 (야간 당직자용).
* **[고급 리포트]**: 월간 Uptime 보고서 및 상세 응답 지연 속도 추이 차트 다운로드 (CSV / PDF).

## 4. 제외 범위 (Out of Scope)

* **모바일 네이티브 앱:** 초기 배포 시 iOS/Android용 독립 앱은 개발하지 않으며, 모바일 크롬/사파리 브라우저의 반응형 레이아웃 뷰포트 지원으로 대체합니다.
* **서버 자원(CPU/메모리) 상세 분석:** 본 플랫폼은 가용성(Ping/URL HTTP Check)에 초점을 맞추며, 에이전트 상세 자원 수집 분석은 1단계 서비스 범위에서 제외하고 URL 헬스체크에 집중합니다.
* **수동 세금계산서 수금 처리:** 모든 요금은 Stripe 및 토스 카드를 통한 정기 구독 결제로만 수납하며, 수동 계좌 이체나 세금계산서 직접 발행은 처리하지 않습니다.

---

## 5. 제품 출시 기준 (Launch Criteria)
* **보안 결함 제로:** 결제 API 및 어드민 제어 API가 비인가 토큰 조건에서 완벽하게 `401 Unauthorized`로 차단되어야 함.
* **알림 성공률 보장:** 내부 모의 장애 발생 테스트 시 카카오 알림톡이 10초 이내에 수신자에 도달해야 함.
* **크로스 브라우저 호환:** Chrome, Safari, Edge 브라우저의 데스크톱 및 모바일 뷰포트에서 대시보드가 깨짐 없이 출력되어야 함.
