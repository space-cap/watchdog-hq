# [초보자 가이드] 깃허브 & 구글 소셜 로그인 (OAuth) 발급 완전 정복 가이드

본 문서는 `watchdog-hq` 서비스에서 **GitHub 소셜 로그인**과 **Google 소셜 로그인**을 실제로 가동하기 위해 필요한 `CLIENT_ID` 및 `CLIENT_SECRET` 키를 발급받는 방법을 초보자의 눈높이에 맞춰 화면 클릭 위치까지 상세하게 설명하는 가이드 문서입니다.

---

## 1. 깃허브(GitHub) 소셜 로그인 키 발급 가이드

깃허브 계정만 있으면 약 2분 만에 무료로 발급받을 수 있습니다.

### [1단계] 깃허브 개발자 설정 페이지 이동
1. 브라우저에서 [GitHub 홈페이지(https://github.com)](https://github.com)에 접속하여 로그인합니다.
2. 오른쪽 맨 위에 있는 **본인 프로필 아이콘**을 클릭합니다.
3. 드롭다운 메뉴 맨 아래에 있는 **`Settings` (설정)** 항목을 클릭합니다.
4. 왼쪽 사이드바 메뉴 맨 아래로 스크롤을 내려 **`Developer Settings`**를 클릭합니다.

### [2단계] OAuth 앱 신규 등록
1. 왼쪽 메뉴에서 **`OAuth Apps`**를 클릭합니다.
2. 오른쪽 상단의 초록색 **`Register a new application`** (또는 `New OAuth App`) 버튼을 클릭합니다.

### [3단계] 필수 정보 입력
화면에 나타나는 4개 입력란에 아래 내용을 정확히 입력합니다:

* **Application name:** `watchdog-hq`
* **Homepage URL:** `http://localhost:3088`
* **Application description:** `SaaS 가용성 관제 포털` (선택 사항)
* **Authorization callback URL (가장 중요!):** 
  ```text
  http://localhost:3088/api/auth/callback/github
  ```
  *(주소 끝에 오타가 없도록 그대로 복사하여 붙여넣으세요.)*

### [4단계] Client ID 및 Client Secret 복사
1. 맨 아래 초록색 **`Register application`** 버튼을 클릭합니다.
2. 생성된 페이지 상단에 보이는 **`Client ID`** (예: `Iv1.a1b2c3d4e5f6...`) 문자열을 복사합니다.
3. 오른쪽의 **`Generate a new client secret`** 버튼을 클릭합니다.
4. 새로 화면에 나타나는 **`Client Secret`** (예: `7a8b9c0d1e2f...`) 문자열을 **즉시 복사**합니다.
   *(이 보안 비밀번호는 페이지를 새로고침하면 다시는 보이지 않으므로 꼭 바로 복사하셔야 합니다.)*

## 2. 구글(Google) 소셜 로그인 키 발급 가이드

구글 Cloud 콘솔에서 발급받는 과정입니다. 약 3분 소요됩니다.

### [1단계] 구글 Cloud 콘솔 접속 및 프로젝트 생성
1. 브라우저에서 [Google Cloud Console(https://console.cloud.google.com/)](https://console.cloud.google.com/)에 접속하여 구글 계정으로 로그인합니다.
2. 화면 맨 위 좌측 상단의 **프로젝트 선택 드롭다운**을 클릭한 뒤, 팝업창 오른쪽 상단의 **`새 프로젝트`**를 클릭합니다.
3. **프로젝트 이름**에 `watchdog-hq`를 입력하고 아래 **`만들기`** 버튼을 누릅니다.
4. 상단 프로젝트 선택 메뉴에서 방금 만든 **`watchdog-hq`** 프로젝트를 선택하여 전환합니다.

### [2단계] OAuth 동의 화면 설정 (최초 1회 필수)
1. 왼쪽 상단 메뉴 버튼(☰)을 누르고 **`API 및 서비스` ➡️ `OAuth 동의 화면`**을 클릭합니다.
2. User Type에서 **`외부`**를 선택하고 아래 **`만들기`** 버튼을 클릭합니다.
3. 기본 정보 3가지만 입력합니다:
   * **앱 이름:** `watchdog-hq`
   * **사용자 지원 이메일:** 본인 구글 이메일 선택
   * **개발자 연락처 정보 (이메일 주소):** 본인 이메일 주소 입력
4. 맨 아래 **`저장하고 계속`** 버튼을 3번 연속으로 눌러 동의 화면 설정을 마칩니다.

### [3단계] 사용자 인증 정보(OAuth 클라이언트 ID) 만들기
1. 왼쪽 메뉴에서 **`사용자 인증 정보`**를 클릭합니다.
2. 상단의 **`+ 사용자 인증 정보 만들기`**를 누르고 **`OAuth 클라이언트 ID`**를 선택합니다.
3. **애플리케이션 유형:** **`웹 애플리케이션`**을 선택합니다.
4. **이름:** `watchdog-hq-web` (기본값 그대로 두어도 됩니다)
5. 화면 아래로 내려와 **`승인된 리디렉션 URI`** 항목을 찾은 후 **`+ URI 추가`** 버튼을 클릭합니다.
6. 아래 주소를 정확하게 입력합니다:
   ```text
   http://localhost:3088/api/auth/callback/google
   ```

### [4단계] Client ID 및 Client Secret 복사
1. 맨 아래 **`만들기`** 버튼을 클릭합니다.
2. 화면에 떠오르는 **`OAuth 클라이언트가 생성됨`** 팝업창에서 다음 두 가지를 복사합니다:
   * **클라이언트 ID:** (예: `123456789-abc...apps.googleusercontent.com`)
   * **클라이언트 보안 비밀:** (예: `GOCSPX-x1y2z3...`)

## 3. `.env.local` 파일에 진짜 키 적용하기

복사하신 키들을 `H:\lee\watchdog-hq\.env.local` 파일의 해당 항목에 붙여넣어 주시면 됩니다.

```env
# -------------------------------------------------------------
# GitHub 소셜 로그인 (GitHub OAuth)
# -------------------------------------------------------------
GITHUB_CLIENT_ID="방금_복사한_깃허브_Client_ID_붙여넣기"
GITHUB_CLIENT_SECRET="방금_복사한_깃허브_Client_Secret_붙여넣기"

# -------------------------------------------------------------
# Google 소셜 로그인 (Google OAuth)
# -------------------------------------------------------------
GOOGLE_CLIENT_ID="방금_복사한_구글_클라이언트_ID_붙여넣기"
GOOGLE_CLIENT_SECRET="방금_복사한_구글_클라이언트_보안_비밀_붙여넣기"
```

## 💡 참고: 실서비스(Vercel 등)로 배포할 때

나중에 상용 서버나 Vercel로 인터넷에 배포한 후에는, 깃허브와 구글 콘솔 설정 화면의 **리디렉션 URI** 자리에 아래와 같이 **실제 웹 사이트 도메인 주소**를 하나 더 추가해주시기만 하면 됩니다!

* **깃허브 추가 리디렉션 URI:** `https://본인도메인.com/api/auth/callback/github`
* **구글 추가 리디렉션 URI:** `https://본인도메인.com/api/auth/callback/google`
