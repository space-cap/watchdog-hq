# [서비스 기획서] watchdog-hq: 한국형 초경량 SaaS 가용성 모니터링 플랫폼

본 문서는 실시간 웹사이트 및 API 가용성을 감시하고 장애 발생 시 카카오톡과 슬랙으로 즉각 알림을 발송하는 상용 SaaS 서비스 `watchdog-hq`의 서비스 기획서(Product Specification Document)입니다.

---

## 1. 서비스 개요 및 목적

### 1.1 서비스 정의
* **서비스명:** watchdog-hq (왓치독 에이치큐)
* **한 줄 정의:** "내 사이트가 멈추면 3초 만에 카카오톡으로 알려주는 초경량 가용성 감시 비서"
* **핵심 솔루션:** 
  웹사이트, API 서버, 쇼핑몰 랜딩 페이지의 접속 상태를 전 세계 분산 노드에서 실시간으로 감시하고, 장애 발생 및 복구 시점에 슬랙, 디스코드, 이메일은 물론 **국내 친화적인 카카오톡 알림톡 및 문자(SMS)**로 실시간 경고를 발송하는 SaaS형 플랫폼.

### 1.2 비즈니스 배경 및 타겟 고객
* **글로벌 서비스의 한계 극복:** UptimeRobot 등 해외 서비스는 영어 인터페이스, 달러 결제, 해외 전송 문자의 높은 유실률, 그리고 무엇보다 **카카오톡 알림 부재**라는 한계가 명확함.
* **타겟 고객군:**
  * **소규모 쇼핑몰 운영자:** 카페24, 스마트스토어 등 사이트 다운이 매출 타격으로 직결되는 1인 셀러.
  * **마케터 및 캠페인 대행사:** 광고 집행 중 트래픽 초과로 랜딩 페이지가 죽는 것을 즉시 감지해야 하는 실무자.
  * **1인 개발자 및 스타트업:** 외주 프로젝트나 자체 베타 서비스 가용성을 저렴하고 확실하게 상시 관제하고 싶은 개발자.

## 3. 핵심 사용자 여정 및 흐름 (User Flow)

### 3.1 회원 가입 및 온보딩
1. 사용자는 소셜 로그인(구글/깃허브)을 통해 3초 만에 가입을 완료하고 대시보드에 진입합니다.
2. 웰컴 가이드를 통해 본인이 운영 중인 대표 사이트의 URL(예: 쇼핑몰 주소)을 등록하여 **무료 등급(Free)** 감시를 시작합니다.

### 3.2 알림 수신처 설정
1. 사용자는 카카오톡 알림을 받기 위해 대시보드에서 본인의 대한민국 휴대폰 번호를 인증하고 알림톡 수신에 동의합니다.
2. 협업 툴 연동을 위해 Slack Incoming Webhook 주소를 등록해 둡니다.

### 3.3 장애 상황 전이 및 경고 수신
1. 모니터링 대상 서버가 트래픽 초과로 응답하지 않음 (Connection Timeout 또는 503 Service Unavailable 발생).
2. 분산 배치된 `watchdog-server` 노드들이 교차 검증을 수행하여 최종 `OFFLINE` 판정을 내립니다.
3. 중앙 `watchdog-hq` 서버의 알림 큐가 작동하여 등록된 **카카오 알림톡**과 **슬랙**으로 즉각 장애 알림을 발송합니다.
   * *알림톡 예시: "[watchdog-hq] 🚨 장애 발생! 등록하신 '내쇼핑몰' 서비스의 접속이 끊어졌습니다. (응답코드: 503, 장애시각: 20:30)"*
4. 대상 서버가 복구되면 다시 동일하게 정상 복구(`ONLINE`) 카카오 알림톡을 발송하여 점주를 안심시킵니다.

## 4. 화면 구조도 (Information Architecture)

SaaS 웹 포털은 직관적이고 반응성 높은 다크 모드 기반의 **싱글 페이지 애플리케이션(SPA)** 스타일로 설계합니다.

```text
[Main Portal Domain]
  |
  |-- (Public) Landing Page: 서비스 소개, 요금제 비교, 회원가입 유도 버튼
  |-- (Public) Sign-in/Sign-up Page: 구글/깃허브 로그인 페이지
  |
  |-- (Private) Control Console (중앙 관제 센터)
  |     |-- Dashboard: 등록된 가용성 카드 그리드 뷰, 실시간 Uptime 상태 표시, 도트 이력 그래프
  |     |-- Alert Settings: 알림 채널 관리 (휴대폰 번호 등록, Slack Webhook 바인딩)
  |     |-- Subscription & Billing: 요금제 변경, Stripe 정기 카드 결제 관리, 결제 영수증 조회
  |
  |-- (Public) Customer Status Page (개별 고객용 공개 가동 현황판)
        |-- status.watchdog-hq.com/user-id 또는 status.my-domain.com
        |-- 일반 방문자들에게 자사 서비스의 실시간 Uptime 및 과거 장애 점검 이력을 이쁜 스킨으로 노출
```

## 5. 비즈니스 실행 마일스톤 (MVP 단계)

* **1단계 (1~2주차):** Next.js 프론트엔드 콘솔 대시보드 마크업 UI 구현 및 PostgreSQL 데이터베이스 스키마 고정.
* **2단계 (3주차):** 구글/깃허브 Auth 로그인 구현 및 Go Checker 모듈 통신 API 인터페이스(노드 보고용 수신 API) 구현.
* **3단계 (4주차):** 국내 메시징 플랫폼(Solapi 등) 연동을 통한 카카오 알림톡 전송 워커 및 Stripe 결제 시스템 연동.
* **4단계 (5주차):** 베타 테스트 가동 및 1인 커뮤니티(생활코딩, 인디해커 등) 타겟 린(Lean) 오픈 베타 마케팅.
